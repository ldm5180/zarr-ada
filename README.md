# zarr-ada

A small, **SPARK-checked** reader for [Zarr v2](https://zarr.readthedocs.io/)
array stores, written in **Ada 2022**. It reads chunked, Blosc-compressed
N-dimensional arrays from a directory store into memory — enough to consume
data written by `zarr-python` / `numcodecs`, such as the SPXW option-chain
stores this was built for.

```ada
declare
   Ask : constant Zarr.F32.Array_Data := Zarr.F32.Load (Store, "ask");
begin
   --  27299 x 149, row-major; 0-based coordinates
   Put_Line (Float'Image (Zarr.F32.Element_At (Ask, [12_345, 77])));
end;
```

## Scope

Supported, because that is what the target data uses:

- Zarr **v2** directory stores (a group of arrays, each `…/<name>/.zarray`).
- **C (row-major)** order, **little-endian** dtypes `<f4`, `<i4`, `<i8`
  (ready-made readers `Zarr.F32`, `Zarr.I32`, `Zarr.I64`; add others by
  instantiating `Zarr.Arrays` with a decoder).
- **Arbitrary rank** (1-D, 2-D, 3-D, … up to `Max_Rank` = 32), regular chunk
  grids with edge-chunk trimming, and the `.` dimension separator.
- The **Blosc** compressor with any inner codec (zstd, lz4, …) and any shuffle —
  decompression is delegated to the system Blosc library, which handles it all.

Out of scope (raises `Unsupported` or is simply not implemented): Zarr v3,
Fortran order, structured/object dtypes, filter pipelines, sharding, and
non-directory stores.

## Dependency: libblosc

Chunk decompression calls `blosc_decompress` from the system **Blosc** library.
There is no Alire crate for it, so install the development package:

```console
sudo apt install libblosc-dev      # Debian/Ubuntu
```

It is linked via `pragma Linker_Options ("-lblosc")` in `src/zarr-blosc.ads`.

## Design: compile-time where it can be, SPARK where it matters

**Compile-time.** The reader is a generic, `Zarr.Arrays`, parameterised by the
element type, its byte size, the expected dtype, the fill value and a decode
function. Instantiating it (see `Zarr.F32`/`Zarr.I32`/`Zarr.I64`) fixes the
dtype and the little-endian byte→value decode at compile time — there is no
per-element run-time dispatch on the dtype.

**SPARK.** The reader is split into a provable core and thin, clearly marked
boundaries:

| Unit | `SPARK_Mode` | Proven |
|------|--------------|--------|
| `Zarr.Codec` (int decoders) | On | AoRTE + the little-endian assembly |
| `Zarr.Indexing` | On | AoRTE **and** that products/offsets never overflow, for **any rank**, via saturating `Long_Long_Integer` arithmetic |
| `Zarr.Codec.Decode_F4` body | Off | float bit-reinterpret is outside the SPARK value model |
| `Zarr.Blosc`, `Zarr.Stores`, `Zarr.Metadata`, `Zarr.Arrays` | Off | the C call, file I/O and JSON parsing |

`gnatprove -P zarr.gpr` proves the core with no unproved checks. The key trick:
`Zarr.Indexing` computes element counts and linear offsets in
`Long_Long_Integer` and *saturates* at a sentinel one past `Natural'Last`, so a
too-large array can never overflow the arithmetic — it just reports a value the
caller rejects. That is what lets the index math be rank-generic *and* fully
proved without a per-dimension overflow precondition.

## Build, test, prove

```console
alr build                                   # build the library
alr test                                    # AUnit suite (hermetic fixture)
alr exec -- gnatprove -P zarr.gpr -j0       # SPARK proof
alr exec -- gprbuild -P example/example.gpr # the spxw example
./example/bin/read_spxw [path/to/store.zarr]
```

The test suite reads a committed Zarr v2/blosc fixture under
`tests/fixtures/` (1-D, 2-D and 3-D arrays, generated by `zarr-python`), so it
needs no external data; run it from the repo root.

## Layout

```
src/      zarr.ads + the reader (codec, indexing, blosc, stores, metadata,
          the generic arrays + F32/I32/I64 instances, fills)
example/  read_spxw.adb — reads a real SPXW store and checks known values
tests/    AUnit suite + committed fixture store (test_zarr.gpr)
```

## Requirements

GNAT + `gprbuild` and Alire; compiles as **Ada 2022**. `libblosc-dev` for
linking; `aunit` for the tests; `gnatprove` for the proof.
